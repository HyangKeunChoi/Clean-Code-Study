# 창발성

## 찰발적 설계로 깔끔한 코드 구현하기

## 창발성(Emergence)

> 하위 계층에는 없는 특성이나 행동이 상위 계층에서 자발적으로 돌연히 출연하는 현상

## 창발적 설계
+ 단순한 4가지를 반복하다보면 전체적으로 깨끗한 코드가 만들어 진다.
1. 모든 테스트를 실행한다.
2. 중복을 없앤다.
3. 프로그래머 의도를 표현한다.
4. 실용적 관점에서 타협한다.

## 모든 테스트를 실행한다.

## 중복을 없앤다.
+ 기존의 코드를 최대한 재활용한다.
+ Template Method 패턴

## 의도를 표현한다.

## 실용적 관점에서 타협한다.
+ 과도한 설계 X
+ DIP (의존성 역전 원칙)
